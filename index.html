Engineering Physics Robot Competition

Key Concepts and Tools Used

  • Circuit prototyping
  • Mechanical prototyping
  • System integration
  • Team collaboration
  • OnShape
  • C++

Description

Over the course of the summer of 2022 I worked in a group of 4 to design, build, and program a robot from scratch. The objective of the robot was to acquire plastic idols along a competition course. The course consisted of main 3 sections, line following, IR tracking, and ziplines. Almost every part of the robot was manufactured in the lab using a variety of tools such as a laser cutter, lathe, milling machine and 3D printer. The robot was controlled using a STM32 Bluepill, along with several other circuits that were constructed by my teammates and I. Several circuits required extensive testing and troubleshooting, where an oscilloscope to identify faulty signals. Upon completing the necassary circuits, I wrote firmware in C++ used to detect idols along the course using sonar sensors, which would then control linear robot arm and claws to reach and grab the idol. Integration of the subsystems, such as acquiring idols while driving, was by far the most challenging and time consuming part of the competition. Nonetheless, our team placed fourth, despite being down a team member for more than half of the build time.

Engineering Physics machine learning competition

Key Concepts and Tools Used

  • Robot Operating System (ROS)
  • OpenCV
  • Immitation learning
  • Data collection
  • Linux
  • Partner collaboration

Description

This project was created for the purpose of developing skills in machine learning and computer vision, using a simulated environment in Gazebo. The robot seen moving around the road, drove around a simulated course while viewing license plates on the cars scattered on the edges of its path. The robot was controlled using an imitation learning model which published ROS command velocities to the robot from a Python script running OpenCV image processing on the robots front camera. The neural net was trained from data that was collected by my teammate and I driving manually around the course. Over 25,000 images were collected, along with data points for each command velocity seen during its respective frame. Another Python script running OpenCV processing on the camera, was attempting to identify contours that resembled the front of the cars. Whenever this occurred, two other imitation models were used to read the license plate and parking spot number. A data pipeline was created using Python such that thousands of images could be captured in different lighting perspectives and orientations, to collect the data used on the license plate identification. The projects directory structure was developed in Linux and managed by my teammate and I using Git.

SolidWorks DeLorean

Key Concepts and Tools Used

  • SolidWorks
  • Team collaboration

Description

During my first year at UBC, I worked in a group of six to design a ‘Back to the Future’ DeLorean using Solidworks. I assembled over 100 parts to create the V6 engine that resided at the back of the DeLorean. This engine had a fully functioning crank and camshaft assembly, with 6 individual pistons and 12 rocker arms to open fuel and exhaust valves in the cylinder heads seen in purple. I worked with my group to complete the Delorean by assembling each group members sub-assemblies. This project enabled my to not only strengthen my SolidWorks skills but also develop my abilities to coordinate and organize with a team to produce a large scale project.
